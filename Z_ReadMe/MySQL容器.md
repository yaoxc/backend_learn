# 创建容器
> docker run -d \
--name mysql \
-e MYSQL_ROOT_PASSWORD=123456 \
-e MYSQL_USER=yaoxc02 \
-e MYSQL_PASSWORD=123456 \
-e MYSQL_DATABASE=cex \
-p 3306:3306 \
mysql:5.7.29


----
## 参数解释

这条 `docker run` 命令用于在Docker中以守护进程模式（`-d`）启动一个MySQL 5.7.29版本的容器，以下是对每个参数的详细解释：

1. **`--name mysql`**：
    - 为正在创建的Docker容器指定名称为 `mysql`。这有助于在Docker环境中轻松识别和管理该容器。例如，在使用 `docker stop mysql` 命令停止容器或 `docker logs mysql` 查看容器日志时，使用这个名称会很方便。

2. **环境变量设置**：
    - **`MYSQL_ROOT_PASSWORD=123456`**：
        - 设置MySQL数据库的 `root` 用户密码为 `123456`。`root` 用户是MySQL中的超级用户，拥有对数据库的所有操作权限。在实际应用中，应使用更复杂、安全的密码。
    - **`MYSQL_USER=yaoxc02`**：
        - 创建一个新的MySQL用户 `yaoxc02`。这个用户将用于日常数据库操作，与 `root` 用户相比，权限可以进行更精细的设置，以增强数据库的安全性。
    - **`MYSQL_PASSWORD=123456`**：
        - 设置新创建的 `yaoxc02` 用户的密码为 `123456`。同样，在生产环境中应使用强密码。
    - **`MYSQL_DATABASE=cex`**：
        - 当容器启动时，会自动创建一个名为 `cex` 的数据库。新创建的用户 `yaoxc02` 可以对这个数据库进行操作，具体权限取决于后续的授权设置。

3. **端口映射 `-p 3306:3306`**：
    - 将主机的3306端口映射到容器内部的3306端口。MySQL默认使用3306端口进行通信。通过这种映射，外部（主机）可以通过访问主机的3306端口来与容器内运行的MySQL服务进行交互。例如，可以使用MySQL客户端连接到 `127.0.0.1:3306`（假设主机IP为 `127.0.0.1`）来访问容器中的MySQL数据库。

4. **`mysql:5.7.29`**：
    - 指定要使用的Docker镜像为 `mysql:5.7.29`，即MySQL数据库的5.7.29版本镜像。如果本地没有该镜像，Docker会自动从Docker Hub等镜像仓库下载。

通过执行这条命令，你可以快速在Docker中启动一个配置好的MySQL数据库实例，方便进行开发、测试等工作。不过，在生产环境中，还需要考虑更多的安全和性能优化措施，如数据持久化、网络隔离等。
