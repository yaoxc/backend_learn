# 盘口显示的买卖盘的列表数据，包不包含市价单？

盘口显示的买卖盘列表数据，**不包含任何市价单**——无论是买盘还是卖盘，仅展示「未成交的限价单（含基于限价逻辑的复杂订单）」，市价单因自身特性不会在盘口停留，具体原因和细节如下：


### 一、核心结论：市价单为何不会出现在盘口？
关键在于「市价单的交易逻辑」与「盘口的功能定位」完全不匹配：
1. **市价单的核心特性：主动成交，无未成交状态**  
   市价单（Market Order）的本质是「放弃价格控制，追求立即成交」：
    - 市价买单：以当前卖盘最优价（卖一）为成交价格，主动吃掉卖盘的限价卖单，完全成交或部分成交后，无剩余（若有剩余，部分CEX会自动转为限价买单，但此时已不是市价单）；
    - 市价卖单：以当前买盘最优价（买一）为成交价格，主动吃掉买盘的限价买单，同理无未成交状态。  
      市价单提交后会直接进入撮合流程，要么瞬间成交，要么转为限价单（剩余部分），不会以「市价单形态」在盘口排队。

2. **盘口的核心功能：展示「被动挂单的潜在对手盘」**  
   盘口（买盘+卖盘）的价值是给用户提供「市场深度」参考：
    - 买盘：展示「未成交的限价买单」（被动挂单，等待卖盘订单匹配），用户挂限价卖单时可判断能否立即成交；
    - 卖盘：展示「未成交的限价卖单」（被动挂单，等待买盘订单匹配），用户挂限价买单时可判断成交可能性。  
      盘口只记录「等待匹配的被动订单」，而市价单是「主动发起匹配的订单」，自然不会出现在盘口中。


### 二、补充说明：特殊场景的边界情况
1. **部分成交的市价单剩余部分**：  
   若市价单因市场流动性不足，仅部分成交（如市价卖单10 BTC，买一仅5 BTC），剩余5 BTC在部分CEX中会自动转为「限价卖单」（成交价格为买一价），此时这部分剩余订单会进入卖盘排队——但此时它已不是市价单，而是限价单，因此盘口显示的仍是限价单形态。

2. **复杂订单类型的盘口显示规则**：  
   无论是止损市价单（Stop Market）、冰山订单、条件订单，只要核心是「市价属性」，均不会出现在盘口；若核心是「限价属性」（如止损限价单），仅在触发条件后转为限价单，才会进入盘口。


### 总结
盘口买卖盘列表的核心是「未成交的限价单集合」，市价单因「主动成交、无排队状态」的特性，不会出现在盘口中。这一规则是所有CEX的统一设计，本质是为了清晰展示市场深度（被动挂单的供需情况），帮助用户判断即时成交可能性。
