> - kafka数据流
### exchange-api → Kafka → exchange-relay → exchange 撮合（简要数据流向）

- **下单链路（新委托）**
  - `exchange-api`：
    - 写入 `exchange_order`、`order_kafka_outbox`（PENDING）。
    - 出库后由 `OrderOutboxService.trySendById` 发送到 Kafka Topic：`exchange-order-ingress`。
  - `exchange-relay`：
    - 监听 `exchange-order-ingress`，按 `symbol` 作为 key 转发到内部撮合 Topic：`exchange-order`。
  - `exchange`（撮合服务）：
    - `ExchangeOrderConsumer` 消费 `exchange-order`，将订单送入对应 `CoinTrader` 进行撮合。

- **撤单链路（取消委托）**
  - `exchange-api`：
    - 撤单请求写入 `order_kafka_outbox`，由 `OrderOutboxService` 发送到 Topic：`exchange-order-cancel-ingress`。
  - `exchange-relay`：
    - 监听 `exchange-order-cancel-ingress`，按 `symbol` 作为 key 转发到内部撤单 Topic：`exchange-order-cancel`。
  - `exchange`（撮合服务）：
    - `ExchangeOrderConsumer` 消费 `exchange-order-cancel`，在 `CoinTrader` 中从订单簿移除对应订单。
