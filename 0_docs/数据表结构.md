# backend_learn 数据表结构说明

> 基于 `sql/init/bizzan.sql` 与当前项目业务（撮合、钱包、充值提币、C2C、合约、活动等）整理。用于新人从表结构入手熟悉业务，或排查问题时快速定位表与字段含义。

---

## 一、业务熟悉入口与推荐阅读顺序

熟悉当前业务时，建议**按下面顺序**从表到代码、从主链路到周边，循序渐进。

### 1. 第一层：基础主数据（谁、什么币、什么交易对）

| 顺序 | 表名 | 用途 | 说明 |
|------|------|------|------|
| 1 | **coin** | 平台支持的币种主数据 | 决定哪些币可充提、转账、交易；与钱包、exchange_coin 关联 |
| 2 | **member** | 用户/会员主表 | 注册、登录、实名、邀请关系、OTC 收款方式等 |
| 3 | **exchange_coin** | 币币交易对配置 | 哪些 symbol 可撮合、手续费、精度、启停、发行活动等；撮合与行情都依赖 |

**建议**：先弄清「用户是谁（member）」「有哪些币（coin）」「有哪些交易对（exchange_coin）」，再看资金与订单。

---

### 2. 第二层：资金与钱包（钱在哪、怎么变）

| 顺序 | 表名 | 用途 | 说明 |
|------|------|------|------|
| 4 | **member_wallet** | 用户各币种钱包（余额/冻结） | 核心：balance 可用、frozen_balance 冻结；与 member、coin 关联；下单冻结、成交扣减、撤单解冻都落在此表 |
| 5 | **member_wallet_history** | 钱包变动流水 | 由触发器或业务写入，用于对账与审计（变更前后 balance/frozen） |
| 6 | **deposit_record** | 充值记录（业务侧） | 链上充值到账后的业务记录，与 wallet 模块、Mongo 充值明细配合理解 |

**建议**：把「下单→冻结→成交→扣冻结、加对手币」和「充值→加 balance」两条线落在 member_wallet + 流水/充值表上。

---

### 3. 第三层：撮合与订单（币币交易主链路）

| 顺序 | 表名 | 用途 | 说明 |
|------|------|------|------|
| 7 | **exchange_order** | 币币委托订单 | 每笔限价/市价单一条记录；status、traded_amount、direction、type；撮合引擎恢复未完成订单也查此表 |
| 8 | （Mongo/ES） | 成交明细、K 线 | 成交明细与 K 线多在 MongoDB 或独立存储，与 exchange_order 通过 order_id 关联 |

**建议**：结合文档「订单创建→撮合→资金入账」流程，看 exchange_order 与 member_wallet 的读写时机。

---

### 4. 第四层：提币与链上相关

| 顺序 | 表名 | 用途 | 说明 |
|------|------|------|------|
| 9 | **withdraw_record** | 提币申请与结果 | 用户提币请求、审核、链上 tx、状态；与钱包扣减、链上广播配合 |
| 10 | **member_address** | 用户充值/提币地址 | 各币种对应地址，由钱包服务生成与校验 |

**建议**：与项目中的 RpcController、PaymentHandler、EthService 对照，理解「提币申请→扣款→签名广播→状态回写」与 withdraw_record 的对应关系。

---

### 5. 第五层：C2C、合约、活动与运营

| 顺序 | 表名 | 用途 | 说明 |
|------|------|------|------|
| 11 | **ctc_order** / **ctc_acceptor** | C2C 法币买卖（如 CNY/USDT） | 用户与承兑商之间的订单与承兑商信息 |
| 12 | **otc_order** / **advertise** | OTC 挂单与广告 | 买卖广告与订单 |
| 13 | **contract_coin** / **contract_order_entrust** | 合约交易对与委托 | 合约业务（若启用） |
| 14 | **activity** / **activity_order** | 活动与活动订单 | 抢购、锁仓等活动及参与记录 |
| 15 | **admin** / **admin_role** / **admin_permission** | 后台管理员与权限 | 运营与风控后台 |

按需阅读，不必一次全看。

## 三、核心表详解（含字段注释）

### 1. coin（币种主数据）

**用途**：平台支持的币种配置，决定哪些币可充值、提现、转账、参与交易；钱包服务按此表为用户创建各币种 member_wallet；提币/矿工费/精度等规则也由此表控制。

| 字段 | 类型 | 说明 |
|------|------|------|
| name | varchar(255) PK | 币种唯一标识，如 USDT、Bitcoin、Ethereum |
| name_cn | varchar(255) | 中文名 |
| unit | varchar(255) | 显示单位，如 USDT、BTC、ETH |
| status | int | 状态：0 正常等 |
| sort | int | 排序 |
| can_recharge | int | 是否可充值 |
| can_withdraw | int | 是否可提现 |
| can_transfer | int | 是否可内部转账 |
| can_auto_withdraw | int | 是否支持自动提现 |
| min_withdraw_amount | decimal(18,8) | 最小提币数量 |
| max_withdraw_amount | decimal(18,8) | 最大提币数量 |
| withdraw_threshold | decimal(18,8) | 提现阈值（如低于此值不可提） |
| withdraw_scale | int | 提币精度（小数位数） |
| min_tx_fee / max_tx_fee | double | 最小/最大矿工费 |
| miner_fee | decimal(18,8) | 矿工费（固定或默认） |
| min_recharge_amount | decimal(18,8) | 最小充值数量 |
| cny_rate / usd_rate | double | 法币汇率 |
| is_platform_coin | int | 是否平台币 |
| enable_rpc | int | 是否启用 RPC（链上交互） |
| cold_wallet_address | varchar | 冷钱包地址 |
| deposit_address | varchar | 充值地址（若统一分配） |
| has_legal | bit | 是否有法币对 |
| information / infolink | varchar | 币种介绍与链接 |
| account_type | int | 币种账户类型 |

### 3. member_wallet（用户各币种钱包）

**用途**：用户在某币种下的可用余额、冻结余额及链上充值地址；下单冻结、成交扣减/增加、撤单解冻、充值到账、提币扣减均作用在此表；唯一约束 (member_id, coin_id) 保证一人一币种一条记录。

| 字段 | 类型 | 说明 |
|------|------|------|
| id | bigint PK | 主键 |
| member_id | bigint | 用户 ID |
| coin_id | varchar FK(coin.name) | 币种 |
| balance | decimal(26,16) | 可用余额 |
| frozen_balance | decimal(26,16) | 冻结余额（挂单占用） |
| address | varchar | 该币种充值地址（由钱包服务生成） |
| is_lock | int | 是否锁定：0 否 1 是 |
| to_released | decimal(18,8) | 待释放数量（锁仓等活动） |
| version | int | 乐观锁版本号，更新时递增防并发 |

---

### 4. exchange_coin（币币交易对配置）

**用途**：定义哪些交易对可撮合、手续费、价格与数量精度、启停、市价买卖开关、最小挂单量等；撮合引擎（CoinTrader）与行情服务都依赖此表；enable=2 表示未在撮合中心启用。

| 字段 | 类型 | 说明 |
|------|------|------|
| symbol | varchar PK | 交易对，如 BTC/USDT |
| base_symbol | varchar | 计价币，如 USDT |
| coin_symbol | varchar | 交易币，如 BTC |
| base_coin_scale | int | 计价币精度 |
| coin_scale | int | 交易币精度 |
| fee | decimal(8,4) | 交易手续费率 |
| enable | int | 是否启用：1 启用 2 停用等 |
| sort | int | 排序 |
| enable_market_buy / enable_market_sell | int | 是否允许市价买/卖 |
| min_sell_price | decimal(18,8) | 最低卖价 |
| flag | int | 0 普通 1 推荐等 |
| min_turnover | decimal(18,8) | 最小挂单成交额 |
| max_volume / min_volume | decimal(18,8) | 最大/最小下单量 |
| max_buy_price | decimal(18,8) | 最高买单价 |
| publish_type | int | 发行类型：1 无 2 抢购 3 分摊等 |
| publish_price / publish_amount | decimal(18,8) | 发行价/发行量 |
| start_time / end_time / clear_time | varchar | 活动开始/结束/清盘时间 |
| exchangeable | int | 是否可交易 |
| visible | int | 前台是否可见 |
| robot_type | int | 机器人类型 |
| enable_buy / enable_sell | int | 是否允许买/卖 |
| max_trading_order | int | 最大同时交易订单数，0 不限制 |
| max_trading_time | int | 委托超时自动下架（秒） |
| zone | int | 区域等 |
| instrument | varchar | 交易类型（如 B2C2） |

---

### 5. exchange_order（币币委托订单）

**用途**：每笔限价/市价委托一条记录；下单时写入，撮合过程中更新 traded_amount、status、completed_time 等；撮合重启时根据未完成订单恢复订单簿。

| 字段 | 类型 | 说明 |
|------|------|------|
| order_id | varchar PK | 订单唯一 ID |
| member_id | bigint | 用户 ID |
| symbol | varchar | 交易对，如 BTC/USDT |
| base_symbol / coin_symbol | varchar | 计价币/交易币 |
| direction | int | 方向：0 买 1 卖 |
| type | int | 类型：0 市价 1 限价等 |
| price | decimal(18,8) | 委托价格（限价单） |
| amount | decimal(18,8) | 委托数量 |
| traded_amount | decimal(26,16) | 已成交数量 |
| turnover | decimal(26,16) | 成交额 |
| time | bigint | 下单时间戳 |
| canceled_time | bigint | 撤单时间戳 |
| completed_time | bigint | 完全成交时间戳 |
| status | int | 状态：0 已取消 1 未成交 2 部分成交 3 完全成交等 |
| use_discount | varchar | 是否使用折扣等 |
| order_resource | int | 订单来源（APP/Web/API 等） |

---

### 6. deposit_record（充值记录）

**用途**：链上充值到账后在业务库的记录，与钱包服务、Mongo 充值明细配合；用于对账与展示用户充值列表。

| 字段 | 类型 | 说明 |
|------|------|------|
| id | varchar PK | 主键（如 UUID） |
| member_id | bigint FK | 用户 ID |
| coin_id | varchar FK | 币种 |
| amount | decimal(19,2) | 充值数量 |
| status | int | 状态：0 处理中 1 成功等 |

---

### 7. withdraw_record（提币记录）

**用途**：用户发起的提币申请及审核、链上广播、到账状态；与 member_wallet 扣减、链上 tx 关联。

| 字段 | 类型 | 说明 |
|------|------|------|
| id | bigint PK | 主键 |
| member_id | bigint | 用户 ID |
| coin_id | varchar FK | 币种 |
| address | varchar | 提币目标地址 |
| total_amount | decimal(18,8) | 申请总数量 |
| fee | decimal(18,8) | 手续费 |
| arrived_amount | decimal(18,8) | 预计到账数量 |
| create_time | datetime | 申请时间 |
| deal_time | datetime | 处理时间 |
| transaction_number | varchar | 链上交易哈希 |
| status | int | 状态：待审核/处理中/成功/失败等 |
| is_auto | int | 是否自动审核 |
| can_auto_withdraw | int | 是否支持自动提币 |
| admin_id | bigint FK | 处理管理员 ID |
| remark | varchar | 备注 |

---

### 8. member_wallet_history（钱包变动流水）

**用途**：记录 balance/frozen_balance 变更前后，用于对账与审计；可由触发器在 member_wallet 更新时自动写入。

| 字段 | 类型 | 说明 |
|------|------|------|
| id | bigint PK | 主键 |
| member_id | int | 用户 ID |
| coin_id | varchar | 币种 |
| before_balance | decimal(18,8) | 变更前可用余额 |
| after_balance | decimal(18,8) | 变更后可用余额 |
| before_frozen_balance | decimal(18,8) | 变更前冻结余额 |
| after_frozen_balance | decimal(18,8) | 变更后冻结余额 |
| op_time | datetime | 操作时间 |

## 四、其余表简要说明（用途 + 关键字段）

- **activity**：活动配置（类型、时间、价格、数量、锁仓/挖矿规则等）；**activity_order**：用户参与活动的订单。
- **admin**：管理员 id、用户名、密码、角色、部门等；**admin_role** / **admin_permission** / **admin_role_permission**：角色与权限；**admin_access_log**：操作日志。
- **advertise**：OTC 广告（类型、币种、价格、数量、支付方式等）；**otc_order**：OTC 订单；**otc_coin**：OTC 支持币种配置。

- **country**：国家中文名、英文名、区号、语言、法币等；**department**：部门名、负责人、备注。
- **contract_coin**：合约交易对；**contract_order_entrust**：合约委托（开平仓、杠杆、保证金等）；**contract_option** / **contract_option_coin** / **contract_option_order**：期权/预测合约及订单。
- **member_address**：用户某币种充值/提币地址；**member_api_key**：API Key、权限、绑定 IP 等；**member_transaction**：用户资金流水类型与金额；**wallet_trans_record**：钱包层交易记录。
- **reward_record**：奖励记录；**reward_activity_setting** / **reward_promotion_setting**：活动/推广奖励配置；**member_bonus**：用户分红；**member_promotion**：推广关系；**member_invite_stastic** / **member_invite_stastic_rank**：邀请统计与排行。
- **locked_order** / **locked_order_detail**：锁仓活动订单及明细；**mining_order** / **mining_order_detail**：挖矿订单及明细。
- **legal_wallet_recharge** / **legal_wallet_withdraw**：法币充值/提现记录；**hot_transfer_record**：热钱包归集记录（金额、冷地址、矿工费、tx 等）。
- **transfer_address** / **transfer_record**：内部转账白名单与记录；**withdraw_code_record**：提币验证码/安全校验记录。
- **red_envelope** / **red_envelope_detail**：红包与领取明细；**sign** / **member_sign_record**：签到配置与用户签到记录；**promotion_card** / **promotion_card_order**：推广卡及订单；**quick_exchange**：闪兑；**website_information**：站点配置；**app_revision**：APP 版本；**tb_sms**：短信记录；**init_plate**：机器人初始盘口参数；**financial_item** / **financial_order**：理财项目与订单；**business_auth_apply** / **business_auth_deposit** / **bussiness_cancel_apply**：商家认证与保证金；**member_level**：等级配置；**member_application** / **member_application_config**：用户应用与配置；**exchange_favor_symbol**：用户收藏交易对；**dividend_start_record**：分红发放记录。
